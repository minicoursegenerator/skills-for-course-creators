# Contributing

Thanks for helping make this directory useful to educators, L&D teams, instructional designers, and course creators.

## What belongs here

A skill should be included only when it supports a real learning workflow:

- Creating course assets such as docs, slides, worksheets, ebooks, visuals, quizzes, or spreadsheets.
- Researching, summarizing, or grounding source material for a lesson.
- Building interactive practice, coaching, role-play, or assessment experiences.
- Analyzing interviews, meetings, learner feedback, or training transcripts.
- Organizing and delivering course production work.

General-purpose skills are welcome only when their learning use case is obvious and practical.

## Submission requirements

Every listing must include:

- Skill name.
- Category.
- Source URL.
- Owner or maintainer.
- One-sentence summary.
- Best-fit audience, such as `L&D`, `educator`, `course creator`, `instructional designer`, `trainer`, or `learning operations`.
- License or `See source` when the upstream license is unclear.
- Install and use instructions in the upstream source.
- A short explanation of the education, training, or course creation use case.

Do not submit duplicate listings. If a similar skill is already listed, explain why the new one is materially different.

## Category rules

Use one primary category:

- `interactive-practice`
- `course-assets`
- `research-source-material`
- `training-analysis`
- `visual-learning`
- `delivery-productivity`

If a skill could fit multiple categories, choose the category where a course creator would most likely look for it.

## Quality bar

Listed skills should be inspectable, documented, and useful without guesswork. Prefer skills that include:

- A clear `SKILL.md`.
- Concrete triggers or use cases.
- Dependencies and setup steps.
- Examples or sample outputs.
- A visible license.
- Recent maintenance or a stable, simple implementation.

Avoid skills that are mostly prompts with no workflow, have unclear ownership, require unsafe credentials handling, or are unrelated to learning work.

## First-party Mini Course Generator skills

First-party MCG skills have a stricter checklist:

- Documented install path.
- Tested use path.
- Examples or archetypes where relevant.
- Clear license.
- No broken relative links in the copied directory content.
- Release notes or source repository link when this directory contains only a copy.

## Pull request checklist

- [ ] I added or updated `catalog.yml`.
- [ ] I updated `README.md` to match the catalog.
- [ ] I checked all new links.
- [ ] I included owner/source and license information.
- [ ] I explained the learning or course creation use case.
- [ ] I avoided duplicate listings.
- [ ] I reviewed any copied first-party skill content for broken relative links.

## Attribution

Always credit the original owner or source repository. This directory links to and describes skills; it does not claim ownership of third-party work.
