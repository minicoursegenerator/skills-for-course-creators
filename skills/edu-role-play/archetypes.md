# Archetypes

v1 ships five archetypes. Each is a complete composition you can copy via `edu-role-play init <name> --archetype <id>`.

| id | When to pick it |
| --- | --- |
| `skeptical-buyer` | Sales rep practicing a discovery call with a guarded, numbers-driven enterprise buyer. |
| `difficult-customer` | Support agent practicing de-escalation with a frustrated customer whose account is broken. |
| `compliance-auditor` | Team lead practicing a GDPR / SOC 2-style interview with an external auditor. |
| `performance-review` | Manager practicing a direct-feedback conversation with an underperforming report. |
| `technical-interviewer` | Candidate practicing system-design explanation against a probing interviewer. |

All five pass `edu-role-play lint`. Use them as starting points, not final artifacts — always swap in the user's product, team, and specific objectives before shipping.
